# 메세지 큐

> 날짜: 2026-03-27

## 내용

메세지 큐는 메세지의 무손실을 보장하는, 비동기 통신을 지원하는 컴포넌트다.

### 1. 기본 아키텍처

- **생산자(producer/publisher)**: 메세지를 만들어 큐에 발행(publish)한다.
- **소비자(consumer/subscriber)**: 큐에서 메세지를 받아 그에 맞는 동작을 수행한다.

생산자는 소비자가 메세지를 처리할 준비가 되어 있지 않아도 메세지를 발행할 수 있고,
소비자는 생산자가 가용한 상태가 아니더라도 메세지를 수신할 수 있다.

### 2. 메세지 큐의 장점

서비스 또는 서버 간 결합이 느슨해져서, 규모 확장성이 보장되어야 하는 안정적 애플리케이션 구성에 좋다.

- **비동기 처리**: 생산자는 큐에 메세지를 넣고 바로 다음 작업을 진행할 수 있다. 소비자는 자기 타이밍에 큐에서 메세지를 꺼내 처리한다.
- **독립적 확장**: 생산자와 소비자가 분리(Decoupling)되어 있으므로, 각각 독립적으로 확장할 수 있다.

### 3. 예시: 사진 보정 애플리케이션

사진 보정은 시간이 오래 걸릴 수 있는 프로세스이므로, 비동기적으로 처리하면 편하다.

웹 서버는 사용자의 보정 요청을 큐에 작업(job)으로 넣고, 바로 응답을 반환한다.
보정 작업 프로세스(worker)는 큐에서 작업을 꺼내 보정을 수행한다.

큐로 인해 웹 서버와 worker는 분리되어 있기 때문에,
사진 보정 요청이 폭발적으로 늘어나더라도 worker만 늘려주면 된다.

**웹 서버 → 큐 → worker, 각각 독립적으로 확장 가능!**
