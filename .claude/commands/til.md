# /til — 대화 → TIL 자동 작성

현재 Claude Code 세션에서 사용자와 주고받은 내용을 정리해 TIL 문서를 만들고 커밋/푸시까지 한 번에 처리한다.

`$ARGUMENTS` 가 비어있으면 대화 컨텍스트에서 주제와 키워드를 직접 추론한다. 비어있지 않으면 그 힌트를 우선 반영한다.

## Step 1 — 카테고리 결정

다음 중 하나에 매핑한다. 디렉토리는 실제로 `ls` 로 확인해서 신뢰할 것 (디렉토리가 추가/변경됐을 수 있음).

```bash
ls -d */ | grep -v '\.claude\|\.github'
```

매핑이 모호하면 후보 2~3개를 사용자에게 한 번 확인받는다. 새 카테고리가 필요해 보이면 반드시 확인.

## Step 2 — 제목 & 파일명 결정

- 제목 형식: `<주제> — <세부 키워드 나열>` (em dash 사용)
- 파일명 형식: `<주제> - <세부 키워드 나열>.md` (일반 hyphen)
- 한글 + 핵심 영어 키워드 혼용 가능
- 예: `JPA 핵심 용어 - fetch, dirty, flush, version, auto-flush.md`

기존 동일/유사 주제가 있으면 덮어쓰지 말고 사용자에게 확인:

```bash
ls <카테고리>/ | grep -i '<핵심키워드>'
```

## Step 3 — 본문 작성

[_template.md](../../_template.md) 구조를 따른다. 핵심 규칙:

- 첫 줄: `# <제목>` (em dash 포함)
- 둘째 줄: `> 날짜: YYYY-MM-DD` (오늘 날짜)
- `## 내용` 아래 키워드별 H3 섹션 (`### <키워드> — <짧은 부연>`)
- **각 H3 섹션 끝에 `> **면접 예상 질문:** <질문>` 한 줄만 추가. 답변은 절대 적지 않는다.**
- 섹션 사이 구분자 `---`
- 마지막에 `## 학습 정리` (불릿 3~5개) + `## 참고` (있으면)

본문 작성 가이드:
- 대화에서 나온 코드 예시, 비유, 질문 흐름을 그대로 살린다
- 사용자가 헷갈려했던 포인트를 명시적으로 짚는다 (그 부분이 면접 질문 후보)
- 외부 링크 URL은 사용자가 명시한 경우에만 추가. 추측해서 만들지 말 것

## Step 4 — 파일 작성

`<카테고리>/<파일명>.md` 경로로 Write.

## Step 5 — 커밋 & 푸시

```bash
git add <파일경로>
git commit -m "feat: <간결한 한 줄 설명> TIL 작성"
git push
```

- 커밋 메시지 형식: `feat: <주제 간결 설명> TIL 작성`
- 현재 브랜치에 그대로 푸시 (브랜치 전환 금지)
- **README.md 는 절대 건드리지 않는다.** GitHub Actions 가 main push 시 자동 업데이트한다.

## 주의사항

- [_template.md](../../_template.md) 자체를 수정하지 말 것
- 면접 예상 질문에 답변을 적지 말 것 (질문만)
- 새 카테고리 디렉토리는 사용자 확인 없이 만들지 말 것
- `git push` 실패 시 (예: 업스트림 미설정) 사용자에게 알리고 멈춘다 — 임의로 force push 하지 말 것
