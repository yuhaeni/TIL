# RAM

> 날짜: 2026-04-06

## 내용

### RAM의 역할과 특성

RAM(Random Access Memory)은 CPU가 실행할 프로그램과 데이터를 임시로 저장하는 주기억장치이다.

- **휘발성(Volatile)**: 전원이 꺼지면 저장된 데이터가 모두 사라진다
- **임의 접근(Random Access)**: 어떤 주소든 동일한 시간에 접근 가능 (순차 접근인 보조기억장치와 대비)
- CPU는 보조기억장치에 직접 접근하지 않고, 반드시 RAM을 거쳐 데이터를 읽고 쓴다

### CPU가 보조기억장치가 아닌 RAM을 거치는 이유

CPU와 보조기억장치(HDD, SSD) 사이에는 큰 속도 차이가 존재한다.

- CPU는 나노초(ns) 단위로 동작하지만, 보조기억장치는 밀리초(ms) 단위로 동작
- CPU가 보조기억장치에 직접 접근하면 데이터를 기다리는 동안 CPU가 놀게 되어 **병목(Bottleneck)** 이 발생한다
- RAM은 보조기억장치보다 훨씬 빠르기 때문에, 실행에 필요한 데이터를 미리 RAM에 올려두고 CPU가 RAM에서 가져가는 구조를 사용한다

이것이 **메모리 계층 구조(Memory Hierarchy)** 의 핵심 원리이다.

| 계층 | 속도 | 용량 | 가격 |
|---|---|---|---|
| 레지스터 | 가장 빠름 | 가장 작음 | 가장 비쌈 |
| 캐시 | 매우 빠름 | 작음 | 비쌈 |
| RAM | 빠름 | 중간 | 중간 |
| 보조기억장치 | 느림 | 큼 | 저렴 |

### RAM 용량과 성능의 관계

RAM 용량이 크면 많은 프로그램을 동시에 올려둘 수 있어 성능이 향상되지만, **무한정 늘린다고 성능이 비례해서 증가하지는 않는다.**

- RAM 용량이 부족하면: 프로그램을 보조기억장치에서 반복적으로 올리고 내려야 하므로 느려짐 (스와핑 발생)
- RAM 용량이 충분하면: 필요한 프로그램이 모두 올라가 있으므로 보조기억장치 접근이 줄어듦
- RAM 용량이 필요 이상으로 크면: 이미 필요한 데이터가 모두 올라가 있으므로 추가 용량의 효과가 미미함

즉, RAM 외에도 **CPU 처리 속도, 디스크 I/O, 네트워크 지연** 등 다른 병목 지점이 존재하기 때문에 RAM만 늘리는 것은 완벽한 해결책이 될 수 없다.

> **면접 예상 질문:** RAM 용량을 무한정 늘리는 것이 성능 향상의 완벽한 해결책이 될 수 없는 이유는?

### 휘발성과 데이터 영속화

RAM의 휘발성은 실무에서 **데이터 정합성** 문제를 일으킬 수 있다.

- 예: 결제 처리 중 서버가 예기치 않게 꺼지면, RAM에 있던 처리 중인 데이터가 사라져 돈은 나갔는데 결제가 완료되지 않는 상황 발생
- 이를 방지하기 위해 중요한 데이터는 **DB 등 비휘발성 저장 장치에 영속화(Persistence)** 한다
- WAL(Write-Ahead Logging), 트랜잭션 로그 등의 기법으로 장애 시에도 데이터를 복구할 수 있도록 설계
