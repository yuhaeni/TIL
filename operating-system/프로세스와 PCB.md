# 프로세스와 PCB

> 날짜: 2026-03-29

## 내용

### 프로세스란?

실행 중인 프로그램을 말한다. 모든 프로세스는 CPU를 사용해야 하지만 CPU 자원은 한정적이기 때문에, 각 프로세스는 할당된 시간 만큼만 CPU를 사용하고 시간이 지나면 다른 프로세스에게 양보한다.

### PCB (Process Control Block)

운영체제는 프로세스의 실행 순서를 관리하기 위해 PCB를 사용한다. 옷의 태그처럼 프로세스를 식별하는 정보를 담고 있다.

- **PID**: 프로세스 식별 아이디
- **프로세스 상태**: 현재 프로세스의 상태
- **메모리 저장 위치**: 프로세스가 메모리 어디에 올라와 있는지
- **레지스터 정보 (PC)**: 다음에 실행할 명령어 위치
- **CPU 스케줄링 정보**: 언제 어떤 CPU를 할당받을지

### 문맥 교환 (Context Switching)

프로세스 A의 CPU 할당 시간이 끝나 프로세스 B로 전환될 때:

1. 프로세스 A는 현재까지의 작업 정보(문맥)를 PCB에 저장
2. 프로세스 B의 문맥 정보를 PCB에서 불러와 이어서 실행

이 과정을 **문맥 교환**이라 한다.

### 프로세스의 메모리 영역

| 영역 | 설명 | 특징 |
|------|------|------|
| 코드 영역 | CPU 명령어 저장 | 읽기 전용 |
| 데이터 영역 | 전역 변수 저장 | 프로그램 실행 동안 유지 |
| 힙 영역 | 프로그래머가 직접 할당 | 사용 후 반환 필수 (미반환 시 메모리 누수 발생) |
| 스택 영역 | 지역 변수, 매개 변수 저장 | 함수 실행 종료 시 자동 소멸 |

- **정적 할당 영역**: 코드 영역, 데이터 영역
- **동적 할당 영역**: 힙 영역, 스택 영역

메모리 내에서 힙은 위→아래, 스택은 아래→위 방향으로 쌓여 주소가 겹치지 않도록 한다.

### 프로세스 상태

```
생성 → 준비 ⇄ 실행 → 종료
               ↕
             대기
```

1. **생성**: 메모리에 적재되어 PCB가 발급된 상태
2. **준비**: CPU 할당을 기다리는 상태
3. **실행**: CPU를 할당받아 실행 중인 상태
4. **대기**: 입출력 장치 완료를 기다리는 상태
5. **종료**: 프로세스가 종료된 상태

### 프로세스 계층 구조

프로세스는 부모-자식 관계로 계층 구조를 형성한다. 자식 프로세스는 또 다른 자식 프로세스를 생성할 수 있어 트리 형태의 구조를 이룬다.
