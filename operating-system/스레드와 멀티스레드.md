# 스레드와 멀티스레드

> 날짜: 2026-03-30

## 내용

### 스레드란?

프로세스의 실행 단위이다. 과거에는 하나의 프로세스에 하나의 스레드만 존재했지만(단일 스레드), 요즘은 하나의 프로세스가 여러 스레드를 가질 수 있어 한 번에 여러 작업을 동시에 처리할 수 있다.

### 스레드가 공유하는 것 vs 공유하지 않는 것

| 구분 | 항목 |
|------|------|
| 공유 O | 코드 영역, 데이터 영역, 힙 영역, 파일 |
| 공유 X | 스택 영역, 프로그램 카운터(PC), 레지스터 |

각 스레드는 **자기만의 실행 흐름**을 가지기 때문에 PC/스택/레지스터는 스레드마다 독립적으로 존재한다.

- **스택이 별도**: 지역 변수는 스레드 간 공유되지 않음
- **힙은 공유**: 동시성 문제(race condition)가 발생할 수 있음

### 멀티 프로세스 vs 멀티 스레드

동일 작업을 여러 번 처리하는 상황을 예로 들면:

| 방식 | 동작 | 단점 |
|------|------|------|
| 멀티 프로세스 | 메모리에 동일한 프로세스를 N개 적재 | 코드/데이터/힙 영역이 N배로 늘어 메모리 낭비 |
| 멀티 스레드 | 하나의 프로세스 안에 스레드 N개 생성 | 하나의 스레드 오류가 프로세스 전체에 영향 |

멀티 스레드는 코드/데이터/힙 영역이 **프로세스 내 하나만 존재**하기 때문에 멀티 프로세스보다 메모리를 효율적으로 사용할 수 있다.

### 멀티 스레드의 단점

멀티 프로세스는 프로세스 간 독립적이라 하나가 죽어도 다른 프로세스에 영향이 없다. 반면 멀티 스레드는 자원을 공유하기 때문에 **하나의 스레드에 문제가 생기면 프로세스 전체에 영향**을 줄 수 있다.
