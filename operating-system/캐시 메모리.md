## 내용

### 참조 지역성과 캐시 적중률

캐시 메모리가 효율적으로 동작하는 근거는 **참조 지역성(Locality of Reference)** 원리이다.

- **시간 지역성(Temporal Locality)**: 최근에 접근한 데이터에 다시 접근하려는 경향. 반복문에서 변수를 재사용하는 것이 대표적 예시
- **공간 지역성(Spatial Locality)**: 접근한 메모리 공간 근처를 접근하려는 경향. 배열을 순회하는 것이 대표적 예시

캐시 적중률이 갑자기 떨어지는 원인도 참조 지역성으로 설명할 수 있다. 기존 로직이 바뀌어 CPU의 메모리 접근 패턴이 달라지면, 반복문 없이 매번 새로운 변수를 사용하게 되어 캐시에 적재된 데이터를 재활용하지 못하고 매번 메모리에 접근하게 된다.

> **면접 예상 질문:** 캐시 적중률이 갑자기 떨어지는 원인을 참조 지역성 관점에서 설명하시오.

### 공간 지역성과 자료구조 선택

공간 지역성은 자료구조 선택에도 영향을 미친다.

- **배열(Array)**: 메모리에 연속적으로 배치 → 하나를 접근하면 인접 데이터가 캐시 라인에 함께 올라옴 → 캐시 히트율 높음
- **연결 리스트(Linked List)**: 각 노드가 메모리 여기저기에 흩어져 있음 → 다음 노드 접근 시 캐시 미스 빈번

백엔드에서 성능이 중요한 상황에서 배열 기반 구조를 선호하는 이유 중 하나이다.

> **면접 예상 질문:** 배열과 연결 리스트의 캐시 성능 차이를 공간 지역성 관점에서 설명하시오.

### 멀티 코어 환경의 캐시 구조

- **L1 캐시**: 코어마다 독립적으로 존재. 가장 빠르고 용량이 가장 작다
- **L2 캐시**: 코어마다 독립적이거나 공유 (아키텍처에 따라 다름)
- **L3 캐시**: 모든 코어가 공유. 상대적으로 느리고 용량이 크다

L1을 코어마다 독립적으로 두는 이유: L1은 가장 빠른 캐시인데, 여러 코어가 공유하면 동시 접근 시 대기가 발생하여 빠른 속도라는 장점이 사라진다. L3는 상대적으로 느리고 용량이 크므로 공유해도 성능 저하가 크지 않다.

#### 분리형 L1 캐시 (L1I / L1D)

- **L1I(Instruction)**: 명령어 전용 캐시
- **L1D(Data)**: 데이터 전용 캐시
- CPU는 명령어 인출과 데이터 접근을 동시에 수행해야 하는데, 같은 캐시를 쓰면 둘이 경합하여 한쪽이 대기해야 한다. 분리하면 동시 접근이 가능하여 속도가 향상된다.

### 캐시 적중률 100%의 트레이드오프

캐시 적중률을 높이려면 더 많은 데이터를 캐시에 저장해야 하므로 캐시 용량을 늘려야 한다. 그러나 캐시 용량을 늘리면:

- **속도가 느려진다**: 용량이 커질수록 탐색 시간이 증가
- **가격이 비싸진다**: 캐시는 SRAM으로 만들어져 RAM(DRAM)보다 훨씬 비쌈

저장 장치 계층 구조의 핵심은 상위 계층일수록 빠르고 작고 비싸다는 점이다. 캐시 메모리의 존재 이유인 "빠른 속도"가 희석되므로, 무조건 적중률 100%를 추구하는 것이 정답은 아니다.

> **면접 예상 질문:** 캐시 메모리의 용량을 무한정 늘려 적중률을 100%로 만들 수 없는 이유는?
