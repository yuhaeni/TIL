# 메모리 주소 공간

> 날짜: 2026-04-07

## 내용

### 논리 주소와 물리 주소

- **논리 주소(Logical Address)**: CPU가 프로그램을 실행할 때 사용하는 주소. 각 프로그램은 0번지부터 시작하는 독립적인 주소 공간을 가진다
- **물리 주소(Physical Address)**: 실제 RAM에서의 주소. 프로그램이 메모리에 적재될 때마다 달라진다
- **MMU(Memory Management Unit)**: 논리 주소를 물리 주소로 변환하는 하드웨어. `물리 주소 = 논리 주소 + 베이스 레지스터`

논리 주소와 물리 주소를 분리하는 이유: 같은 프로그램이라도 실행할 때마다 물리 주소가 달라지기 때문에, CPU가 물리 주소를 직접 사용하면 프로그램끼리 서로의 메모리를 덮어쓰는 문제가 발생한다. 각 프로그램에게 독립적인 논리 주소 공간을 부여하고 실행 시점에 변환함으로써 이를 해결한다.

### 베이스 레지스터와 한계 레지스터

- **베이스 레지스터(Base Register)**: 프로그램이 메모리에 적재된 시작 물리 주소를 저장. 논리 주소 변환의 기준점
- **한계 레지스터(Limit Register)**: 프로그램이 사용할 수 있는 최대 논리 주소 범위를 저장. 이 값을 초과하는 접근을 차단

**접근 흐름:**

1. CPU가 논리 주소로 접근 시도
2. 논리 주소 ≥ 한계 레지스터 값이면 → 인터럽트(트랩) 발생, 운영체제가 프로세스 강제 종료
3. 논리 주소 < 한계 레지스터 값이면 → `물리 주소 = 논리 주소 + 베이스 레지스터`로 변환 후 접근

한계 레지스터가 없으면 프로그램이 자기 범위를 넘어 다른 프로그램의 메모리를 읽거나 덮어쓸 수 있다. 이것이 **버퍼 오버플로우(Buffer Overflow)** 취약점의 핵심 원리이다.

> **면접 예상 질문:** 한계 레지스터가 없다면 어떤 보안 문제가 발생할 수 있는가?

### 세그멘테이션 폴트(Segmentation Fault)

CPU가 한계 레지스터보다 높은 논리 주소에 접근하려 하면:

1. 인터럽트(트랩) 발생
2. 운영체제가 해당 프로세스를 강제 종료

리눅스에서 발생하는 `Segmentation fault` 에러가 바로 이 상황에 해당한다.

> **면접 예상 질문:** CPU가 한계 레지스터보다 높은 논리 주소에 접근하려 할 때 어떻게 처리되는가?

---

### 외부 단편화와 페이징의 등장

베이스/한계 레지스터 방식은 프로그램 전체를 메모리의 **연속된 공간**에 통째로 올려야 한다.

프로그램들이 올라오고 내려가는 과정에서 빈 공간이 띄엄띄엄 생기면, 총 빈 공간은 충분해도 연속되지 않아 프로그램을 적재하지 못하는 **외부 단편화(External Fragmentation)** 문제가 발생한다.

이 한계를 극복하기 위해 프로그램을 고정 크기(페이지)로 잘라 비연속적으로 적재하는 **페이징(Paging)** 기법이 등장했다.

> **면접 예상 질문:** 베이스/한계 레지스터 방식의 한계는 무엇이고, 이를 극복하기 위해 어떤 기법이 등장했는가?
