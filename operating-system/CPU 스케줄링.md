# CPU 스케줄링

> 날짜: 2026-03-30

## 내용

### CPU 스케줄링이란?

모든 프로세스는 CPU를 필요로 하고, 먼저 할당받고 싶어 한다. 운영체제는 모든 프로세스에게 공평하게 CPU를 할당해야 하는데, 이 역할을 **CPU 스케줄링**이라 한다.

### 프로세스 우선순위

프로세스마다 우선순위가 다르며, 운영체제는 각 프로세스의 PCB에 우선순위를 명시한다.

| 종류 | 설명 | 우선순위 |
|------|------|----------|
| 입출력 집중 프로세스 | 입출력 작업이 많은 프로세스 | 높음 |
| CPU 집중 프로세스 | 연산 작업이 많은 프로세스 | 낮음 |

입출력 집중 프로세스를 먼저 실행하면, CPU를 할당받자마자 대기 상태로 빠지기 때문에 CPU가 노는 시간 없이 CPU 집중 프로세스를 이어서 실행할 수 있다. → **CPU 활용률을 높일 수 있다.**

### 스케줄링 큐

운영체제가 모든 프로세스의 PCB를 일일이 탐색하는 건 비효율적이다. 그래서 **스케줄링 큐**에 프로세스를 담아 관리한다. 순서대로 꺼내되, 그 중에서도 우선순위가 높은 프로세스에 CPU를 먼저 할당한다.

- **준비 큐**: CPU를 필요로 하는 프로세스들이 대기
- **대기 큐**: 입출력 장치를 필요로 하는 프로세스들이 대기

### 선점형 vs 비선점형 스케줄링

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 선점형 | 운영체제가 자원을 강제로 빼앗아 다른 프로세스에 할당 가능 | 자원을 고르게 분배 | 잦은 컨텍스트 스위칭으로 오버헤드 발생 |
| 비선점형 | 프로세스가 자원을 사용 중이면 다른 프로세스에 할당 불가 | 오버헤드가 적음 | 자원 독점 가능, 기아 현상 발생 위험 |

대부분의 운영체제는 **선점형 스케줄링**을 사용한다.

- **컨텍스트 스위칭 오버헤드**: 전환이 너무 잦으면 프로세스 실행 시간보다 문맥 저장/복원 시간이 더 길어져 CPU가 정작 해야 할 일을 못 할 수 있다.
- **기아 현상(Starvation)**: 비선점형에서 우선순위가 낮은 프로세스가 CPU를 오래 점유하면, 우선순위가 높은 프로세스가 계속 대기하는 상황이 발생한다.
