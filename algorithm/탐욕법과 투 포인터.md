# 탐욕법과 투 포인터

> 날짜: 2026-04-14

## 내용

### 탐욕법 (Greedy)

매 순간 최선의 선택을 하여 전체 최적해를 구하는 알고리즘이다.

구명보트 문제에서의 적용: 가장 무거운 사람은 어차피 짝을 찾기 어려우므로, 가장 가벼운 사람과 먼저 매칭을 시도한다. 매 순간 "무거운 사람 + 가장 가벼운 사람" 조합을 선택하는 것이 전체 보트 수를 최소화하는 최선의 선택이다.

Greedy가 성립하는 조건: 각 단계의 최선 선택이 전체 최적해로 이어질 수 있어야 한다. 반례가 존재하면 Greedy가 아닌 DP 등 다른 방법이 필요하다.

### 투 포인터 (Two Pointer)

정렬된 배열에서 두 개의 인덱스(포인터)를 양 끝에서 이동시키며 조건을 만족하는 쌍을 탐색하는 기법이다.

**적용 조건:**
- 배열이 **정렬**되어 있어야 함 (양 끝 비교가 의미 있으려면 정렬이 전제)
- 두 값의 합/차 비교 문제

**이동 규칙 (구명보트):**

```
left = 0 (가장 가벼운 사람)
right = people.length - 1 (가장 무거운 사람)

합 > limit  → 무거운 사람 혼자 탑승 → right만 감소
합 <= limit → 둘 다 탑승 → left 증가, right 감소
```

**종료 조건:** `left > right` (두 포인터가 엇갈릴 때)

**주의:** `while (left < right)`를 쓰면 홀수 명일 때 마지막 1명을 놓친다. 반드시 `while (left <= right)`를 사용해야 한다.

```java
Arrays.sort(people);
int left = 0, right = people.length - 1;

while (left <= right) {
    if (people[left] + people[right] <= limit) {
        left++;
        right--;
    } else {
        right--;
    }
    answer++;
}
```

### 투 포인터를 쓸 수 있는 유형

| 유형 | 설명 |
|---|---|
| 두 수의 합 (Two Sum) | 정렬 후 양 끝에서 합이 target인 쌍 찾기 |
| 연속 부분 배열 | 조건을 만족하는 구간의 최소/최대 길이 찾기 |
| 중복 제거 | 정렬된 배열에서 유니크한 원소만 남기기 |

공통점: **정렬된 배열** + **두 값의 관계(합, 차, 범위)** 를 비교하는 문제에서 O(n²)을 O(n)으로 줄일 수 있다.

## 학습 정리

- Greedy는 매 순간 최선의 선택이 전체 최적해로 이어질 수 있을 때 사용
- 투 포인터는 정렬된 배열에서 두 값의 합/차 비교 문제에 적용하여 O(n²) → O(n) 달성
- `while (left <= right)` 종료 조건으로 홀수 명의 마지막 원소 누락 방지
- 혼자 타든 같이 타든 매 반복마다 보트 카운트를 증가시켜야 함
