# Arrays.sort() 내부 동작

> 날짜: 2026-04-10

## 내용

### int[] vs Integer[] 정렬 알고리즘 차이

`Arrays.sort()`는 배열 타입에 따라 서로 다른 정렬 알고리즘을 사용한다.

| 구분 | `int[]` | `Integer[]` |
|---|---|---|
| 정렬 알고리즘 | DualPivotQuicksort | TimSort |
| 속도 | 빠름 | 느림 (boxing 오버헤드) |
| 안정 정렬 | X | O |
| Comparator | 불가 | 가능 |

기본 타입 배열은 값 자체를 비교하면 되므로 빠른 Quicksort 계열을 사용하고, 객체 배열은 안정 정렬이 필요한 경우가 있으므로 TimSort를 사용한다.

> **면접 예상 질문:** `Arrays.sort(int[])`와 `Arrays.sort(Integer[])`는 같은 정렬 알고리즘을 사용하는가? 차이점은?

---

### 안정 정렬(Stable Sort)

**안정 정렬**: 같은 값을 가진 요소들의 원래 순서가 정렬 후에도 유지되는 정렬.

```
정렬 전:    [{철수, 3}, {영희, 3}]
안정 정렬:  [{철수, 3}, {영희, 3}]  // 순서 유지
불안정 정렬: [{영희, 3}, {철수, 3}]  // 순서 바뀔 수 있음
```

`int` 원시값은 값 자체만 존재하므로 순서가 바뀌어도 의미가 없다. 반면 `Integer` 객체는 Heap에 서로 다른 주소로 존재하며, 추가 속성(이름, 순위 등)이 있을 수 있어 원래 순서가 의미를 가질 수 있다.

점수가 같은 학생을 정렬할 때 "점수가 같으면 입력 순서 유지" 같은 조건이 있다면 안정 정렬이 필수다.

> **면접 예상 질문:** Java가 `int[]`에는 불안정 정렬, `Integer[]`에는 안정 정렬을 사용하는 이유는?

---

### Comparator와 기본 타입

Comparator는 Object만 받을 수 있기 때문에 `int[]`에는 Comparator를 적용할 수 없다. `int`는 기본 타입이라 객체가 아니기 때문이다.

`int[]`를 내림차순으로 정렬하는 두 가지 방법:

1. `Integer[]`로 박싱 후 Comparator 사용 → boxing/unboxing 오버헤드로 성능 저하
2. `int[]` 그대로 오름차순 정렬 후 인덱스 역방향 접근 (권장)

```java
Arrays.sort(arr); // 오름차순 정렬
for (int i = arr.length - 1; i >= 0; i--) {
    // 역방향 접근 = 내림차순 효과
}
```

### O(n log n) 시간복잡도

- `n` = 입력 크기
- `log n` = n을 2로 몇 번 나눌 수 있느냐 (예: `log₂8 = 3`)
- `n log n` = 매 단계마다 n번씩 비교하며 log n 단계를 반복

**n=8일 때:**

| 단계 | 비교 횟수 |
|---|---|
| 1단계 | 8번 |
| 2단계 | 8번 |
| 3단계 | 8번 |
| **총계** | **8 × 3 = 24번** |

DualPivotQuicksort와 TimSort 모두 평균 시간복잡도는 O(n log n)이다.

> **면접 예상 질문:** O(n log n) 시간복잡도란 무엇을 의미하는가?

### Quicksort 최악의 경우와 DualPivotQuicksort

Quicksort의 최악 시간복잡도는 **O(n²)** 이다.

pivot이 매번 가장 작은(또는 가장 큰) 값으로 선택되면, 한쪽 구간은 0개, 다른 쪽은 n-1개가 되어 균등 분할이 안 된다. **이미 정렬된 배열**에 단순 Quicksort를 적용하면 이 최악 케이스가 발생한다.

**DualPivotQuicksort의 개선:**

pivot을 2개 사용하여 구간을 3개(왼쪽, 중간, 오른쪽)로 분할한다. 한쪽에 몰릴 확률이 낮아져 균등 분할 가능성이 높아지고, 실제 성능이 일반 Quicksort보다 빠르다.

> **면접 예상 질문:** Quicksort의 최악 시간복잡도가 O(n²)이 되는 경우는? DualPivotQuicksort는 어떻게 이를 개선했는가?

## 학습 정리

- `int[]`는 DualPivotQuicksort(불안정), `Integer[]`는 TimSort(안정) 사용
- 안정 정렬은 동일 값 요소의 원래 순서를 보장하며, 객체 배열에서 의미를 가짐
- 기본 타입 배열에 Comparator 적용 불가 → 내림차순은 역방향 인덱스 접근이 효율적
- Quicksort 최악은 O(n²), DualPivot은 pivot 2개로 분할하여 이를 개선
